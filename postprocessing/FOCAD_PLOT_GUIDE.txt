CellFoundry FA Plot Guide
=========================

This guide explains the plots generated by:
  python postprocessing/fa_phase_report.py

Files and meaning
-----------------

1) fa_metrics_trends_<tag>.png
- Left axis: attached_ratio = attached FAs / total FAs.
- Right axis: mean_f_mag = average traction magnitude per FA [nN].
- Main use: global stability and trend check over time.

2) fa_metrics_trends_smoothed_<tag>.png
- Same two metrics as above, with rolling averages overlaid.
- Faint curves = raw data, thick curves = smoothed trend.
- Main use: detect long-timescale drift and compare noisy runs.

3) fa_polarity_attached_ratio_<tag>.png
- front_attached_ratio and rear_attached_ratio are plotted together.
- Also plots (front - rear) to show asymmetry directly.
- Main use: validate polarity effect (front persistence vs rear turnover).

4) fa_polarity_kon_<tag>.png
- Effective attachment rates (k_on_eff) for front and rear groups.
- Main use: confirm front/rear attachment-rate bias is active.

5) fa_polarity_koff_<tag>.png
- Effective baseline detachment rates (k_off_0_eff) for front and rear groups.
- Main use: confirm front/rear detachment-rate bias is active.

CSV outputs
-----------
- fa_metrics_<tag>.csv: per-step global FA metrics.
- fa_polarity_<tag>.csv: per-step front/rear split metrics.
- fa_summary_<tag>.csv: compact run summary (final-20% and polarity stats).

Interpretation notes
--------------------
- These diagnostics are model outputs, not direct biological measurements.
- Check NaNs/abrupt jumps first when validating new mechanics.
- Front-vs-rear behavior can shift with geometry, initialization, and parameter set.
